# 图片目录统一与 Anbernic 封面兼容设计

## 目标

刮削产生的新封面统一使用 ROM 文件名存放；可选地在每次刮削前迁移已有索引封面，并可额外生成不进入索引的 Anbernic `Imgs` 封面副本。

## 配置与界面

“刮削设置”新增两个互相独立的全局选项：

- `强制保持图片目录统一`：默认开启。控制是否在刮削前迁移已有索引封面。
- `兼容 Anbernic 封面`：默认关闭。控制是否额外生成 `Imgs` 封面副本。

两个选项写入 `config.json`，并传入批量刮削、右键补全、右键刷新和手动搜索使用的统一刮削入口。

关闭“强制保持图片目录统一”只会跳过已有封面的迁移；本次新产生的封面仍使用标准目录。Anbernic 兼容选项不依赖目录统一选项。

## 标准命名

所有命名均以 ROM 文件名去除最后一个扩展名后的结果为基础，再使用现有文件名清理规则处理非法字符。

- 标准封面：`media/<ROM文件名>/boxfront.<原扩展名>`
- Anbernic 副本：`Imgs/<ROM文件名>.<原扩展名>`

ZIP 游戏以 ZIP 文件名作为 ROM 文件名。图片保留原始编码与扩展名，不进行 JPEG 转码。

标准封面路径继续写入 Pegasus 的 `assets.boxFront` 和 `gamelist.xml` 的 `image` 字段。`Imgs` 路径永远不写入这两个索引。

## 刮削流程

1. 收集 ROM 文件及其展示文件名。
2. 当开启目录统一或 Anbernic 兼容中的任一选项时，读取已有 `metadata.pegasus.txt` 与 `gamelist.xml`，按其中的游戏文件字段匹配 ROM 并解析已索引封面。
3. 若开启目录统一，迁移已索引封面，并更新所有实际存在的索引文件中的封面路径；是否勾选“生成 Pegasus/gamelist”不影响已有索引的迁移。
4. 若开启 Anbernic 兼容，将已有的可用封面复制到 `Imgs`。此步骤独立于目录统一，因此关闭目录统一时也能从旧索引路径复制封面。
5. 执行正常刮削。补全模式对“完整游戏”的跳过发生在媒体整理之后，因此被跳过的游戏仍能完成旧封面迁移和 Anbernic 副本生成。
6. 新下载或从 ROM 提取的封面写入标准目录。
7. 若开启 Anbernic 兼容，将新封面额外复制到 `Imgs`。标准封面仍保留并继续写入索引。
8. 按现有选项创建或更新 Pegasus 与 gamelist 索引。

## 已有封面迁移

迁移仅以 Pegasus 和 gamelist 中明确记录的封面路径为依据，不扫描或猜测无索引图片的归属，也不扫描 `Imgs`。

同一 ROM 同时出现在两个索引时，两个索引都更新为同一个标准相对路径。若两个索引指向不同且都存在的源图片，优先使用 Pegasus 记录的图片；只有 Pegasus 图片不存在时才使用 gamelist 图片。

迁移时先确保目标目录存在，再移动源文件。若目标已存在，则保留目标文件，不覆盖已有标准封面；源文件与目标内容相同时可删除重复源文件，内容不同时保留源文件并记录冲突日志。源文件成功移动或确认重复后，从源文件所在目录开始向上清理空目录，但不会删除游戏根目录、`media`、`Imgs` 或任何非空目录。

路径无法解析、源图片不存在、目标写入失败或索引格式损坏时，记录日志并继续处理其他游戏。只有成功迁移或已存在有效目标时才改写对应索引路径，避免产生失效引用。

## 索引更新

Pegasus 与 gamelist 的现有备份行为保持不变。迁移阶段仅修改封面字段，不改变游戏名称、描述、视频或其他元数据。

正常刮削写回单个游戏时，新路径使用 ROM 文件名计算，不再使用刮削标题或英文标题计算图片目录。

未勾选 Anbernic 兼容的后续刮削不会删除旧的 `Imgs` 文件，因为这些文件不在索引中，也不属于目录统一迁移的扫描范围。

## 测试

新增自动化测试覆盖：

- ROM 文件名生成标准 `media` 与 `Imgs` 路径，且保留图片扩展名。
- 旧 Pegasus 封面迁移并更新字段。
- 旧 gamelist 封面迁移并更新字段。
- 即使未勾选生成对应索引，已有索引仍会迁移。
- 补全模式中的完整游戏在被跳过前完成迁移。
- 关闭目录统一时不移动旧封面，但新封面仍写入标准目录。
- 开启 Anbernic 兼容时保留标准封面并生成副本，索引只记录标准路径。
- 补全模式跳过完整游戏时，仍从其已有索引封面生成 Anbernic 副本。
- 关闭目录统一但开启 Anbernic 兼容时，从旧索引路径复制封面且不改写索引。
- 关闭 Anbernic 兼容时不删除既有 `Imgs` 文件。
- 成功迁移后仅删除空的旧目录；失败时不改写为无效路径。
