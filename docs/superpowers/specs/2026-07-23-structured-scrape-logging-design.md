# 刮削结构化日志设计

## 目标

每次刮削开始即输出完整且脱敏的配置；刮削全过程使用单一、明确的业务前缀，让用户从日志中看出文件扫描、游戏解析、在线搜索、图片下载、图片迁移、索引更新和结束统计。

## 启动配置

`batch_scrape()` 在扫描目录前逐行输出 `[刮削配置]`，覆盖平台、游戏目录、刮削模式、目标范围、文件格式、在线补全、数据源、凭据配置状态、语言、翻译、视频、文件名标题、线程、Pegasus、gamelist、目录统一、Anbernic 兼容、代理和手动搜索词。

API Key 与密码只显示“已配置/未配置”。代理若包含认证信息，只显示协议、主机和端口。

## 前缀

每条日志只使用一个业务前缀，状态写入正文：

- `[刮削配置]`
- `[文件扫描]`
- `[游戏解析]`
- `[游戏搜索]`
- `[图片下载]`
- `[图片整理]`
- `[Anbernic封面]`
- `[元数据补全]`
- `[翻译]`
- `[视频]`
- `[Pegasus]`
- `[gamelist]`
- `[刮削]`
- `[刮削完成]`

移除 `[DEBUG]`、`[DEBUG-search]`、`[DEBUG-boxart]`、`[在线]`、`[封面]`、`[OK]`、`[提示]`、`[跳过]`、`[失败]` 等含义不明确或纯状态前缀。

## 图片整理日志

图片整理开始时记录选项、待检查游戏数及索引存在情况。逐游戏记录：未发现索引、源图缺失、路径已标准、迁移源和目标、目标冲突、Pegasus/gamelist 路径变更、Anbernic 副本、旧文件删除、空目录删除和异常。结束时输出检查、迁移、已标准、Anbernic 复制、缺失、冲突、失败、空目录清理等统计。

## 其他流程日志

游戏解析记录序号、ROM 名称、解析结果和异常；游戏搜索记录搜索词、数据源、回退数据源和匹配结果；图片下载记录 URL、下载结果、保存路径或内置图标；翻译、视频、元数据合并和两个索引写入均记录明确结果。

## 测试

自动化测试覆盖配置完整性、敏感信息脱敏、单前缀格式、图片迁移明细与汇总、Anbernic 复制日志、搜索/下载/索引前缀以及原有刮削行为不回归。
